api.tagger.dk

Via denne URL kan du benytte Taggers REST-API.

API'et understøtter GET og POST, og returnerer JSON

Forespørgsler skal sendes til http://api.tagger.dk/v1/tag (eksempel)

Parametre af typen (ja/nej) skal have værdien "true" eller "false". Den understregede værdi er standardindstillingen.

Parametre af typen (vocab_ids) skal være en kommasepararet liste af ordbogs-id'er. Standardindstillingen er angivet efter lighedstegnet.

Parametre

  • text (tekststreng): Tekst der skal tagges.
    Bemærk: Meget korte tekster får ofte uventede og forkerte nøgleord.

eller

  • url (URI): Internetside hvis indhold skal tagges.

Valgfri parametre

  • keyword (ja/nej): Slå nøgleordsgenkendelse til/fra.
  • k_vocab_ids (vocab_ids=16): Ordbøger der skal bruges til nøgleordsgenkendelse.
  • named_entity (ja/nej): Slå egennavnsgenkendelse til/fra.
  • n_vocab_ids (vocab_ids=13,15,17): Ordbøger der skal bruges til egennavnsgenkendelse.
  • linked_data (ja/nej): Returnér links (URI'er) til relevant info.

Ordbøger (vocab_ids)
  • Personer (id=13)
  • Steder (id=15)
  • Organisationer (id=17)
  • Nøgleord (id=16)

Returdata

En forespørgsel besvares (hvis den blev behandlet fejlfrit) med et JSON-objekt på følgende form: 

{
    "ordbogsnavn": {
        "tag-id": {
            "name": "Tagnavn",
            "rating": relevansscore,
            "matches": ["Synonym for tag",
                        "Genitivform af tag" ...],
            "linked_data" : { "dataudbyder": "URI" ... }
        }
        ... flere tags
    }
    ... flere ordbøger
}
  • "name" er det kanoniske navn for tagget (dvs. personen, stedet, nøgleordet...)
  • "matches" er formerne af tagget fundet i den givne tekst (forekommer kun for egennavne)
  • "rating" er et tal (0-100%) der angiver taggets vigtighed og relevans i teksten
    Bemærk: Tags er som udgangpunkt sorteret efter rating, men da JSON objekter per definition ikke har rækkefølge anbefales det at du selv sorterer de modtagne data.
  • "linked_data" (valgfri) Et objekt med links til relevant data

Er man i tvivl kan det anbefales at lave nogle testforespørgsler, og se på det returnerede data.

Eksempel

GET-forespørgslen: http://api.tagger.dk/v1/tag?url=information.dk&n_vocab_ids=13,17&linked_data=true finder personnavne og organisationer på siden information.dk, og bestemmer ydermere nøgleord relateret til indholdet på siden. Såfremt de forekommer i databasen medsendes også relaterede info-links.
